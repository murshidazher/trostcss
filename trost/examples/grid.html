Layout Breakpoints

You could make breakpoints to change the Grid layout by targeting specific screen sizes. Add `col-[*]-sm`, `col-[*]-md`, `col-[*]-lg`, `col-[*]-xl` * indicates the col-span you need for specific breakpoints.

Col 4
Col 4
Col 4
            <button class="btn" disabled>...</button>

Innate Responsive Grid Layout

If you are lazy and don't want to specify breakpoints, you could use the responsive columns by just adding `col-[*]--res`. At lg and xl -> 12 col system, at md -> 8 col system and sm and xs -> 4 col system

System

Grid Break

If a row exceeds the 12-column per row layout, the system by default shifts the excess column to the next line vertically.

If you want to override the default 12-column per row layout system and want all columns in the same row without a break to next line, then use row--fluid and flex--wrap-nowrap classes.

Add disbaled attribute to make a button unusable and un-clickable. 

              <button class="btn" disabled>...</button>

Types

Labled Icon

Button can contain both a label and icon by adding btn--icon class to the button. 

                  <div class="btn btn--icon">
  <svg>...</svg>
  <span>Button</span>
</div>

Button Icon

Button can contain only icon by adding btn--icon and btn--square or add btn--circle class to determine the shape of the button. 

                  <div class="btn btn--icon btn--square">
  <svg>...</svg>
</div>
<div class="btn btn--icon btn--circle">
  <svg>...</svg>
</div>
